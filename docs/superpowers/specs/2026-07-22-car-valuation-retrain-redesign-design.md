# 二手车估值项目重训与科研控制台改造设计

## 1. 背景与目标

当前项目已经具备 FastAPI 后端、PyTorch 回归模型、质量门禁、估值历史和 LLM 助手的基本骨架，但还不能作为完整的技术项目展示：训练数据和原始训练脚本缺失，现有模型 `R2=-0.015`、10% 误差命中率约 `11.8%`，前后端中文编码损坏，公开接口与训练单位不透明，价格区间也不是统计置信区间。

本次改造的目标是形成一条可复现、可解释、可演示的闭环：

1. 自动下载并记录公开数据来源，清洗为项目统一数据契约。
2. 用固定随机种子完成训练、验证、测试和基线对比，保存模型、预处理器、特征配置、指标和数据溯源信息。
3. 让质量门禁决定模型是否可以被标记为可用，避免把低质量结果包装成准确率。
4. 修复前后端编码、单位和错误处理问题。
5. 将前端改造成“科研答辩型 / Research Console”，突出数据契约、训练版本、质量门禁、指标和估值解释链。

本次不承诺把印度公开数据伪装成中国二手车市场模型。主数据集来自印度市场，价格单位为 INR；界面和报告必须明确显示这一限制。

## 2. 已确定的方案

### 2.1 数据源与合规边界

主数据源使用 `chandanverma07/DataSets` 中的 `car details v4.csv`：

`https://raw.githubusercontent.com/chandanverma07/DataSets/master/car%20details%20v4.csv`

该数据集约 2,059 条记录，含品牌、车型、价格、年份、里程、燃油、变速箱、城市、颜色、车主、发动机排量和座位数等字段，和现有特征契约的重合度最高。下载脚本保存 URL、抓取时间、文件 SHA-256、原始行数和转换版本；原始 CSV 不提交 Git，默认落在本地数据目录并由 `.gitignore` 排除。数据源页面和仓库归属写入 `backend/data/README.md`，正式使用前仍需人工确认上游仓库的许可和再分发条款。

数据中没有事故历史，也没有稳定的车型类别字段。因此：

- `accident_history` 映射为 `unknown`，不能在页面中暗示该字段是真实观测值。
- `vehicle_type` 统一映射为 `car`，并在特征报告中标记为低信息量字段；不把 `Drivetrain` 错当作车身类型。
- `Owner` 的 First/Second/Third/Fourth 映射为 1/2/3/4，其他值为缺失并统一填充为 `unknown` 或中位数，规则写入数据适配器测试。
- `Engine` 解析 `1198 cc` 等文本为数值升，解析失败保留缺失并由训练预处理填充。
- `Price` 保留为 INR，`Kilometer` 保留为 km；不使用未经记录的汇率转换。

### 2.2 训练流水线

新增可从 `backend` 模块运行的训练入口：

```powershell
D:/car-valuation/venv/Scripts/python.exe -m scripts.train_model --download
```

流水线按以下顺序执行：下载并校验原始文件、转换到统一 CSV、去除价格/年份/里程等关键字段无效行、去除完全重复记录、固定随机种子切分 train/validation/test、拟合预处理器、训练回归模型、与均值基线比较、计算指标、执行质量门禁、原子写入模型产物。

模型继续使用现有 PyTorch MLP + sklearn 预处理器的兼容形态，避免破坏后端加载器；训练脚本负责完整生成现有的 `price_mlp.pt`、`preprocess.joblib` 和 `feature_config.json`。数值特征使用中位数填充和标准化，类别特征使用缺失填充和 one-hot，未知类别必须在推理时可处理。目标值不做隐式货币或单位转换。

训练指标至少包含样本数、MSE、RMSE、MAE、R2、10% 误差命中率、均值基线指标、数据切分比例、随机种子、训练时间、数据源和目标货币。测试集指标只在测试集上计算，均值基线必须使用训练集均值，避免评估泄漏。

质量门禁默认要求：测试集 `R2 > 0`、10% 误差命中率 `>= 0.50`，并且 RMSE 不高于均值基线 RMSE。任一条件不满足，仍保存产物和完整指标，但 API 返回 `quality_gate=fail`、`model_status=experimental`，前端不得展示“准确率达标”。阈值由 `metrics.json` 的 `quality_gate` 字段记录，不能散落在前端。

### 2.3 后端契约

新增数据和模型元信息服务，至少提供：数据源名称、抓取时间、样本数、价格单位、特征版本、训练时间、切分方式和质量门禁结果。`/api/metrics` 与 `/api/model-health` 返回这些元信息，使前端不需要硬编码训练状态。

`/api/predict` 的价格字段按模型元数据返回，并增加 `currency` / `price_unit` 字段；区间继续命名为参考区间，不称为置信区间。若质量门禁失败，预测仍可用于研究演示，但响应说明必须包含数据范围和实验性质。模型文件缺失、元信息不完整、输入包含无法处理的字段时返回明确的 503/422，不回退到手写规则价格。新增 `GET /api/model-card` 返回数据集元信息、类别选项、特征说明、货币和当前模型版本，供前端动态渲染。

输入界面保留中文标签，但品牌、城市、车型等选项必须来自 `/api/model-card` 或允许自由输入，不能继续硬编码中国品牌。变速箱、燃油类型等枚举在界面显示中文标签、提交源数据对应的规范值。表单补齐排量、座位数、车主次数、燃油类型、车辆类型、颜色和事故历史字段；事故历史默认 `unknown` 并明确提示“公开数据未提供该字段”。统一由一个适配器完成到模型字段的映射。输入校验、单位换算、默认值和未知类别处理都要有单元测试。历史记录保存货币和模型版本，避免不同训练版本的价格混在一起比较。

### 2.4 前端方向 A：科研答辩型 Research Console

重写 `frontend/src/App.vue` 的损坏文案和页面结构，保留 Vue 3、Vite、Chart.js，不引入不必要的组件库。整体采用浅色中性背景、细边框、低圆角、墨黑正文和绿色状态强调色，采用研究工作台而不是营销式卡片堆叠。

首屏信息层级如下：

1. 顶部显示项目名称、模型版本、数据集货币和服务状态。
2. 首屏状态带显示质量门禁、测试集 R2、MAE、10% 命中率和数据样本数；失败状态必须高可见并带原因。
3. 左侧为估值输入表单，右侧为结果、参考区间、输入摘要和模型说明。
4. 下方为“模型证据”区域：指标对比图、特征契约、数据来源和限制说明；表单示例值也来自当前数据源的类别统计。
5. LLM 助手定位为可追溯的解释面板：展示检索来源、结构化估值工具调用结果和回答，不伪装成普通闲聊窗口。
6. 历史记录定位为估值实验日志，显示时间、车型、输入摘要、价格、货币和模型版本。

页面必须有加载、空数据、接口失败、模型门禁失败、LLM 未配置和移动端窄屏状态。数字和货币格式由 API 元信息驱动；不要在模板里固定写人民币“万元”。图表只消费后端指标，不根据旧模型文件推断状态。

## 3. 数据流与模块边界

```text
公开 CSV
  -> download_public_dataset.py
  -> public_dataset_adapter.py
  -> normalized_training.csv
  -> train_model.py
  -> model artifacts + metrics/provenance
  -> FastAPI metadata / predict / health
  -> Research Console
```

模块职责保持单一：数据下载器只处理来源和校验；适配器只负责字段转换；训练脚本只负责拟合和产物写入；指标服务只负责加载结构化元数据；模型服务只负责请求适配和推理；前端只负责展示后端事实和收集输入。任何源数据字段映射变更，都必须同时更新契约文档和测试。

## 4. 错误处理与可追溯性

- 网络下载失败：不覆盖现有产物，给出 URL 和本地缓存路径。
- CSV 字段变化：适配器在转换前失败，并列出缺失字段。
- 清洗后样本不足：训练中止，不写入新产物。
- 质量门禁失败：保存实验产物，但保留旧的可运行产物，除非显式传入覆盖参数。
- 前端请求失败：显示原因和重试入口，不生成伪造结果。
- LLM 不可用：保留数值估值和证据面板，解释面板显示未配置状态。

每次训练用临时目录写入并在所有校验通过后替换产物，`metrics.json` 必须能单独解释当前模型来自哪个数据源、哪个切分和哪个特征版本。

## 5. 测试与验收标准

后端新增测试覆盖：原始字段转换、发动机排量解析、Owner 映射、缺失字段拒绝、未知类别推理、训练切分无泄漏、均值基线计算、质量门禁和元信息响应。保留现有契约测试并修复其中的乱码文案。

验收命令：

```powershell
D:/car-valuation/venv/Scripts/python.exe -m unittest discover -s backend/tests
D:/car-valuation/venv/Scripts/python.exe -m scripts.train_model --download
D:/car-valuation/venv/Scripts/python.exe -m scripts.evaluate_model data/normalized_training.csv
cd frontend
npm run build
```

还要启动前后端完成一次浏览器冒烟测试：填写表单、提交估值、查看质量门禁、切换历史/助手模块，确认桌面和移动宽度没有乱码、重叠或横向溢出。

最终交付必须满足：能从空数据目录重建训练数据和模型；页面显示真实数据来源及 INR 单位；质量门禁与指标一致；旧模型质量差时不再被包装；训练失败不会破坏可运行服务；Git 提交中不包含原始下载数据、密钥和临时设计 companion 文件。

## 6. 明确不做的事情

本阶段不接入付费数据、不声称中国市场泛化能力、不把缺失事故字段填成“无事故”、不增加新的大模型训练、不制作营销型首页，也不通过调低质量阈值来让模型强行通过门禁。后续若要面向中国市场，需要新的合规数据源、币种和区域分布验证，并应作为独立数据版本重新评估。
